# Qubit 3D Simulator — 이해 가이드

## 이 시뮬레이터는 무엇을 보여주는가

**큐빗(qubit)의 상태가 어떻게 생겼는지**, 그리고 **게이트가 그 상태를 어떻게 바꾸는지**를 3D로 시각화합니다.

양자컴퓨터의 계산 능력은 **중첩(superposition)** 과 **얽힘(entanglement)** 이라는 두 가지 현상에서 나옵니다. 이 시뮬레이터는 그 두 현상을 눈으로 직접 확인할 수 있게 만든 도구입니다.

---

## 고전 비트 vs 큐빗

고전 컴퓨터의 비트는 항상 0 또는 1, 두 가지 상태 중 하나입니다.

큐빗은 다릅니다. **측정하기 전에는** 0과 1이 동시에 섞인 상태로 존재할 수 있습니다. 이 "섞인 정도"를 구 위의 한 점으로 나타낸 것이 **블로흐 구(Bloch Sphere)** 입니다.

---

## 블로흐 구 읽는 법

화면의 구와 화살표가 큐빗의 현재 상태입니다.

| 화살표 위치 | 의미 |
|------------|------|
| 북극 ↑ | 확실히 \|0⟩ — 측정하면 반드시 0 |
| 남극 ↓ | 확실히 \|1⟩ — 측정하면 반드시 1 |
| 적도 | 0과 1이 정확히 50:50 중첩 |
| 그 사이 | 한쪽으로 치우친 중첩 |
| 화살표 짧음 | 얽힘 상태 — 이 큐빗만으로는 상태를 정의할 수 없음 |
| 화살표 없음 | 완전한 얽힘 — 개별 큐빗의 상태가 존재하지 않음 |

우측 패널의 **State Vector**는 같은 정보를 수치로 보여줍니다.
- α (alpha): \|0⟩ 방향의 복소수 진폭
- β (beta): \|1⟩ 방향의 복소수 진폭
- 확률: 측정 시 해당 상태가 나올 확률 (|진폭|²)

---

## 게이트란 무엇인가

게이트는 큐빗의 상태를 바꾸는 연산입니다. 블로흐 구 위에서 보면 **회전**으로 나타납니다.

| 게이트 | 동작 | 비유 |
|--------|------|------|
| **X** | 북극 ↔ 남극 뒤집기 | 고전 비트의 NOT |
| **Z** | 적도 위에서 180° 위상 회전 | 북극/남극은 그대로 |
| **H** | 북극 → 적도 (또는 적도 → 북극/남극) | 중첩 만들기 |
| **Y** | X+Z 결합 회전 | |
| **S** | Z축 기준 90° 위상 회전 | |
| **T** | Z축 기준 45° 위상 회전 | |
| **RX / RY** | 각각 X, Y축 기준 90° 회전 | |

---

## 측정이 왜 확률적인가

`Measure` 버튼을 누르면 화살표가 북극 또는 남극으로 **순간 튀어버립니다**.

- 화살표가 북극에 가까울수록 → 0이 나올 확률이 높음
- 화살표가 적도에 있으면 → 0과 1이 각각 50% 확률
- 측정 후 상태는 0 또는 1로 확정되고, 이전 중첩 정보는 사라짐

이것이 양자역학의 **파동함수 붕괴**입니다. 측정 행위 자체가 상태를 바꿉니다.

---

## 2큐빗 — 얽힘의 시작

### 벨 상태 (Bell State)

**H → CNOT** 순서로 적용하면 두 큐빗이 얽힙니다.

```
초기:   Q0=|0⟩, Q1=|0⟩          → 화살표 둘 다 북극
H 적용: Q0=중첩, Q1=|0⟩         → Q0 화살표 적도로 이동
CNOT:   두 큐빗이 얽힘            → 화살표 둘 다 사라짐
```

화살표가 사라진다 = 각 큐빗이 **혼자서는 어떤 상태인지 정의할 수 없는 상태**가 된 것입니다.

이 상태에서 Measure를 누르면 **항상 `00` 또는 `11`** 만 나옵니다. Q0가 0이면 Q1도 반드시 0, Q0가 1이면 Q1도 반드시 1 — 두 큐빗이 아무리 멀리 떨어져 있어도 한쪽을 측정하는 순간 다른 쪽의 결과가 즉시 확정됩니다. 아인슈타인이 "유령 같은 원격 작용(spooky action at a distance)"이라고 불렀던 현상입니다.

### CNOT 게이트

제어 큐빗(Control)이 \|1⟩일 때만 타겟 큐빗(Target)을 반전시킵니다.

| Control | Target 전 | Target 후 |
|---------|-----------|-----------|
| \|0⟩ | 변화 없음 | 변화 없음 |
| \|1⟩ | \|0⟩ | \|1⟩ |
| \|1⟩ | \|1⟩ | \|0⟩ |

중첩된 Control 큐빗에 CNOT을 적용하면 얽힘이 생성됩니다.

---

## 3큐빗 — GHZ 상태

**H → CNOT(Q0→Q1) → CNOT(Q0→Q2)** 를 적용하면 세 큐빗이 모두 얽힙니다.

```
|GHZ⟩ = (|000⟩ + |111⟩) / √2
```

세 화살표가 모두 사라지고, Measure를 누르면 **항상 `000` 또는 `111`** 만 나옵니다. 세 큐빗 중 어느 하나를 측정하는 순간 나머지 둘의 결과도 즉시 확정됩니다.

이 상태는 양자 통신, 양자 오류 정정, 양자 텔레포테이션의 핵심 재료로 사용됩니다.

---

## 이 시뮬레이터의 한계

이 시뮬레이터는 큐빗 1~3개의 **순수 상태(pure state)** 만 다룹니다.

| 한계 | 이유 |
|------|------|
| 노이즈 없음 | 실제 양자컴퓨터는 주변 환경과 상호작용하며 오류 발생 |
| 큐빗 3개 이하 | 4큐빗부터 상태공간(2⁴=16)이 커져 블로흐 구로 표현 불가 |
| 얽힘 시각화 제한 | 얽힌 큐빗은 개별 블로흐 구로 완전히 표현할 수 없음 (화살표 축소가 부분적 표현) |

실제 양자컴퓨터(IBM, Google 등)는 수십~수백 큐빗을 사용하며, 그 상태공간은 고전 컴퓨터로는 시뮬레이션 자체가 불가능한 수준입니다.

---

## 핵심 정리

> 측정 전 큐빗은 어딘가에 "있는" 것이 아니라, 가능한 모든 상태의 **확률적 중첩**으로 존재합니다.
>
> 여러 큐빗이 얽히면 개별 상태가 사라지고, 전체 시스템만이 하나의 상태를 가집니다.
>
> 양자컴퓨터는 이 중첩과 얽힘을 동시에 활용해, 고전 컴퓨터가 순차적으로 처리해야 할 계산을 병렬로 수행합니다.
