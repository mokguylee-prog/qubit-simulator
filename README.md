# Qubit 3D Simulator

이 프로젝트는 양자역학의 큐빗 상태를 블로흐 구 형태로 3D 시각화하는 웹 기반 시뮬레이터입니다.

## 실행 방법

`script.js`가 ES 모듈(`import`)을 사용하기 때문에 **파일을 직접 열면 동작하지 않습니다.** 반드시 로컬 서버를 통해 실행해야 합니다.

```bash
python3 -m http.server 8080
```

실행 후 브라우저에서 `http://localhost:8080` 으로 접속하세요.

## 사용법

1. 게이트 버튼(X, Y, Z, H, S, T, RX, RY)을 눌러 큐빗 상태를 변환합니다.
2. 블로흐 구 위의 화살표가 현재 상태를 가리킵니다.
3. `Measure` 버튼으로 측정을 시뮬레이션합니다 (결과에 따라 상태가 붕괴됩니다).
4. `Custom Rotation`에서 축과 각도를 직접 지정해 회전할 수 있습니다.
5. 마우스로 블로흐 구를 자유롭게 회전·확대할 수 있습니다.

## 기능

- X, Y, Z, H, S, T 게이트 적용
- RX(π/2), RY(π/2) 회전
- 사용자 정의 축 회전
- 상태의 복소수 진폭(α, β)과 측정 확률 표시
- 블로흐 구 각도(θ, φ) 실시간 표시
- 3D 블로흐 구 자동 회전 (OrbitControls로 수동 조작 가능)

## 기술 스택

## 주요 변경 이력

### 버그 수정 (2025-05-17)

**블로흐 구가 렌더링되지 않는 문제 수정**

- **원인**: Three.js를 UMD 전역 스크립트(`<script src>`)로 로드하면서 OrbitControls만 ESM으로 import하면 THREE 인스턴스가 두 개 생성되어 충돌이 발생합니다.
- **수정**: `importmap`을 도입해 `three`와 `three/addons/`를 동일한 ESM 소스로 해석하도록 통일했습니다. `script.js` 내 모든 THREE 참조가 단일 인스턴스를 사용합니다.

**화살표 방향 오류 수정**

- **원인**: `lookAt()`은 객체의 +Z축을 목표 방향으로 정렬하는데, 화살표는 +Y축으로 제작되어 있어 실제 블로흐 벡터와 다른 방향을 가리켰습니다.
- **수정**: `setFromUnitVectors(Y축, 블로흐 벡터)`로 교체하고, 블로흐 구의 극축(z)을 Three.js의 Y축(위)으로 매핑했습니다. 이제 |0⟩은 북극(위), |1⟩은 남극(아래)을 정확히 가리킵니다.

**궤도 원 표시 오류 수정**

- **원인**: 두 궤도 원 모두 동일한 회전(`rotation.x = π/2`)이 적용되어 같은 평면(XY)에 겹쳐 있었습니다.
- **수정**: 하나는 적도(XZ 평면), 하나는 자오선(YZ 평면)으로 분리해 블로흐 구의 기준면을 올바르게 표시합니다.
